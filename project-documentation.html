Project Documentation

The “Revolussion” of Renzo Renzi
Information Science and Cultural Heritage – University of Bologna (2024–2025)

1. Project Overview

The “Revolussion” of Renzo Renzi is a Digital Humanities project focused on the archival, bibliographic, and audiovisual materials related to Renzo Renzi (1919–2004), a central figure in the history of film culture in Bologna.

Renzi’s work as critic, filmmaker, researcher, and curator created an analogue network of interrelated objects—books, drawings, photographs, interviews, film materials—that anticipates the logic of Linked Open Data. The project aims to model, encode, and publish a selection of these materials using cultural heritage standards, TEI/XML, RDF, and a web interface.

2. Project Goals

The project follows the official requirements of the course Information Science and Cultural Heritage, and aims to:

  • Select a coherent cultural domain (Renzo Renzi and the Renzo Renzi Collection).
  • Choose at least 10 heterogeneous items (the project includes 15 items).
  • Reuse institutional metadata standards from libraries, archives, and museums.
  • Describe each item through a theoretical model (natural-language analysis of its features and context).
  • Develop a conceptual model reusing existing ontologies and vocabularies (no new ontology is created).
  • Produce:
    • CSV metadata for each item
    • a TEI-encoded text and XSLT transformation
    • CSV → RDF transformation in Python
    • a unified RDF dataset in Turtle
    • a website presenting data, methods, and outputs

Museum / Artistic / Visual Items

  • Scheda OA (ICCD) – drawings, caricatures

Audiovisual / Film

  • FIAF Cataloguing Rules

5. Theoretical Model (Natural-Language Analysis)

Each object was analysed following the descriptive logic of its reference standard, identifying:

  • intrinsic features (title, date, format, technique, extent…)
  • roles and actors (creators, performers, contributors…)
  • relationships to other entities (about, depicts, documents…)
  • institutional context (holding institution, collection, location…)
  • authority control (VIAF, Wikidata, ISIL, ISNI…)

The domain includes:

  • People (Renzo Renzi, Federico Fellini, Giulietta Masina, Nino Rota…)
  • Works (book, screenplay, documentary, film, soundtrack…)
  • Images (photographs, caricatures, drawings…)
  • Events (interviews, premieres, documentary production…)
  • Places (Bologna, Cinema Fulgor, Po River Delta…)
  • Institutions (Renzo Renzi Library, Cineteca di Bologna, ICCU…)

These narratives form the theoretical model that underpins the conceptual model and the RDF representation.

6. Conceptual Model (Ontological Reuse)

The conceptual model does not introduce a new ontology. Instead, it reuses existing vocabularies to describe entities and relationships in a coherent way.

Ontologies and Vocabularies

  • Dublin Core Terms (DCTerms)
  • Schema.org
  • FOAF
  • SKOS
  • RiC-O (conceptual reference for archival logic)
  • CIDOC CRM (conceptual reference for museum logic)
  • IFLA LRM (bibliographic thinking)

Core Classes

  • schema:CreativeWork
  • schema:ImageObject
  • schema:VideoObject / schema:Movie
  • schema:MusicRecording
  • schema:Person, schema:Organization, schema:Place
  • foaf:Agent
  • dcterms:Location

Core Properties

  • dcterms:creator, dcterms:subject, dcterms:date
  • schema:about, schema:locationCreated
  • schema:hasPart, dcterms:hasPart
  • foaf:depicts

The conceptual model is summarized in a Grafoo-style diagram included in the documentation (e.g. docs/conceptual_model.png).

7. Data Production (CSV)

A dedicated CSV file was created for each of the 15 items. In addition, two global files organise the semantic layer:

  • rrr_entities.csv – entities of the domain (people, places, works, concepts…)
  • rrr_triples.csv – relations between those entities

All identifiers follow a snake_case convention and a shared project prefix (rrr:).

8. TEI Encoding & XSLT Transformation

One item, related to La Strada, was encoded using TEI P5. The encoding focuses on a selected sequence and aims to:

  • represent structure, speakers, and relevant segments
  • highlight editorial and interpretative choices

The stylesheet tei2html_lastrada.xsl transforms the TEI file into an HTML edition that is integrated into the project website (file html/lastrada.html).

9. RDF Dataset

The RDF dataset (ttl/rrr.ttl) is generated from the CSV files using Python and RDFLib.

build_rrr_rdf.py

  • reads the metadata from item CSVs and from rrr_entities.csv and rrr_triples.csv
  • constructs a graph using RDFLib
  • serializes the result as Turtle (rrr.ttl)

compare_ttl.py

  • compares two Turtle serializations
  • helps track differences between RDF versions during development
